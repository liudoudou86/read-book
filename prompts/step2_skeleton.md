# 骨架搭建指导

## 什么是骨架？

每一本书都有一个隐藏的骨架结构。作为分析阅读者，你的责任是找出这个骨架——即作者如何组织他的思想、观点和论证。

## 为什么要搭建骨架？

- 理解书的整体结构
- 把握作者的论述逻辑
- 为后续深入分析打基础

## 搭建骨架的三步

### 第一步：一句话概括

用一句话，最多一小段话，概括整本书在讲什么。

**公式**：
> 这本书是关于[主题]，作者认为[核心观点]，目的是让读者[获得什么]。

**示例**：
> 《如何阅读一本书》是关于阅读方法的书，作者认为阅读需要分层次（基础阅读→检视阅读→分析阅读→主题阅读），目的是帮助读者提升理解力和阅读效率。

### 第二步：列出重要部分

将全书的重要部分列举出来，说明它们是如何组成一个整体的。

**公式**：
> 作者将全书分成了 N 个部分：
> - 第一部分在说 [主题]
> - 第二部分在说 [主题]
> - 第三部分在说 [主题]
> 
> 第一个主要部分又可分为：
> - 第一章 [主题]
> - 第二章 [主题]

### 第三步：找出作者要问的问题

每一本书的作者在写作时都带着一个问题或一连串的问题。找出这些问题，就是找到了书籍的「核心驱动」。

**常见问题类型**：
- 这本书要解决什么问题？
- 作者想要回答什么问题？
- 作者想要达成什么目的？

## 输出格式

```markdown
## 书籍骨架

### 一句话概括
[用一句话概括整本书]

### 结构拆解
- 第一部分：[主题] - [核心内容简述]
- 第二部分：[主题] - [核心内容简述]
- 第三部分：[主题] - [核心内容简述]
- ...

### 重要章节
- 第X章：[章名] - [这章解决什么问题]

### 作者要解决的问题
1. 主要问题：...
2. 次要问题：...
3. ...
```

## 实践技巧

1. **先看目录**：目录是作者给的骨架地图
2. **看章节标题**：每个标题往往概括了一章的核心
3. **快速翻阅**：用 5-10 分钟快速浏览，不用细读
4. **不要追求完美**：骨架可以后续修正
